# Test Plan

## トリガーテスト

### Should trigger（発動すべきケース）
- "Geminiにも確認して"
- "Geminiでチェックして"
- "セカンドオピニオンが欲しい"
- "このコードをGeminiにレビューしてもらって"
- "Geminiの意見も聞きたい"
- "別のAIにも見てもらって"

### Should NOT trigger（発動すべきでないケース）
- "このコードをレビューして"（Gemini言及なし → code-reviewer スキルが処理）
- "Gemini CLIをインストールして"（インストール支援、レビューではない）
- "Geminiとは何ですか"（概念説明の質問）
- "コードの問題を直して"（バグ修正依頼）

---

## 機能テスト

### テスト 1: コードファイルのレビュー
1. テスト用コードファイルを用意
2. "このファイルをGeminiにもレビューしてもらって" と指示
3. 期待結果:
   - `gemini -p "..." -m gemini-3-pro-preview` コマンドが実行される
   - Gemini の回答が取得される
   - Claude の見解と統合されたフィードバックが返される

### テスト 2: 429 エラー時のフォールバック
1. `gemini-3-pro-preview` が 429 を返す状況をシミュレート
2. 期待結果: `gemini-2.5-pro` に自動フォールバックして再実行

### テスト 3: 機密ファイルの確認
1. `.env` や認証情報を含むファイルを指定
2. 期待結果: 外部送信前にユーザーへ確認プロンプトが表示される

### テスト 4: CLI 未インストール時
1. `gemini` コマンドがない環境でスキルを実行
2. 期待結果: `npm install -g @google/gemini-cli` のインストール案内が表示される
