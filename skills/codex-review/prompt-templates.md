# Codex 日本語プロンプトテンプレート

`/codex:review` 等のプラグインコマンドに渡す日本語プロンプト集。

---

## コードレビュー

```
以下のコードをレビューしてください：
- バグや潜在的な問題
- パフォーマンスの改善点
- セキュリティの懸念
- コードの可読性・保守性
日本語で回答してください。
```

---

## 論文・学術文章チェック

```
以下の文章を学術的な観点からレビューしてください：
- ロジックの一貫性
- 文法・表現の改善点
- 引用や根拠の妥当性
日本語で回答してください。
```

---

## セキュリティ監査

```
以下のコードをセキュリティの観点から監査してください：
- 認証・認可の問題
- インジェクション攻撃（SQLi, XSS等）への脆弱性
- 機密情報の取り扱い
- 依存ライブラリのリスク
日本語で回答してください。
```

---

## アーキテクチャレビュー

```
以下のコードのアーキテクチャをレビューしてください：
- 設計パターンの適切さ
- モジュール間の依存関係
- スケーラビリティの懸念
- テスタビリティ
日本語で回答してください。
```

---

## PR前の最終確認

```
このPR前の変更について総合的にレビューしてください。
マージしても問題ないか、懸念点があれば日本語で教えてください。
```

---

## 敵対的レビュー（Adversarial Review）

`/codex:adversarial-review` に追加フォーカスとして渡す場合：

```
1. この設計判断は本当に最適か？代替案とそのトレードオフを具体的に示せ
2. この実装が壊れるシナリオは何か？障害モードを列挙せよ
3. 暗黙の前提条件は何か？それが崩れた場合どうなるか
4. スケーラビリティ・保守性・セキュリティで見落としている懸念はないか
5. このアプローチに対する最も強い反論は何か
日本語で回答してください。
```

---

## 構造化出力レビュー（Structured Output）

```
以下の変更をレビューし、結果を以下のJSON形式で返してください：
{
  "verdict": "approve | needs-attention",
  "summary": "総合所見（1-2文）",
  "findings": [
    {
      "severity": "critical | warning | suggestion",
      "title": "問題の短いタイトル",
      "body": "詳細説明",
      "file": "ファイルパス（不明な場合は null）",
      "line": "行番号または範囲（不明な場合は null）",
      "confidence": "high | medium | low",
      "recommendation": "推奨する修正方法"
    }
  ],
  "next_steps": ["次にすべきアクション1", "アクション2"]
}
JSONのみを返してください。説明文は不要です。
```
