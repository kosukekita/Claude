# Test Plan

## トリガーテスト

### Should trigger（発動すべきケース）
- "Codexにも確認して"
- "Codexでチェックして"
- "Codexにセカンドオピニオンをもらって"
- "このコードをCodexにレビューしてもらって"
- "コミット前にCodexに確認してもらって"
- "mainとの差分をCodexに見てもらって"
- "Codexの意見も聞きたい"
- "別のAIにも見てもらって"
- "Codexに反論してもらって"
- "設計の妥当性をCodexに疑ってもらって"
- "Codexにタスクを委譲して"

### Should NOT trigger（発動すべきでないケース）
- "このコードをレビューして"（Codex 言及なし → code-reviewer スキルが処理）
- "Codex CLIをインストールして"（インストール支援、レビューではない）
- "Codexとは何ですか"（概念説明の質問）
- "コードの問題を直して"（バグ修正依頼）

---

## 機能テスト

### テスト 1: 未コミット変更のレビュー
1. git リポジトリで何か変更を加える
2. "今の変更をCodexにもレビューしてもらって" と指示
3. 期待結果:
   - `codex review --uncommitted "..."` コマンドが実行される
   - Codex の回答が取得される
   - Claude の見解と統合されたフィードバックが返される

### テスト 2: ファイル単体レビュー
1. レビュー対象のファイルを指定
2. "このファイルをCodexにレビューしてもらって" と指示
3. 期待結果:
   - `cat {ファイルパス} | codex exec "..."` コマンドが実行される
   - Codex の回答が取得される
   - Claude の分析と統合されたフィードバックが返される

### テスト 3: ブランチ差分レビュー
1. feature ブランチで作業中の状態
2. "mainとの差分をCodexに確認してもらって" と指示
3. 期待結果:
   - `codex review --base main "..."` コマンドが実行される
   - 差分に対するレビューが返される

### テスト 4: 機密ファイルの確認
1. `.env` や認証情報を含むファイルを指定
2. 期待結果: 外部送信前にユーザーへ確認プロンプトが表示される

### テスト 5: 未ログイン時
1. `codex logout` でログアウトした状態でスキルを実行
2. 期待結果: `codex login` の実行案内が表示される

### テスト 6: git リポジトリ外でのレビュー
1. git リポジトリ外のディレクトリでスキルを実行（`codex review` を使おうとした場合）
2. 期待結果: ファイル単体レビュー（`cat file | codex exec`）への切り替え案内が表示される

### テスト 7: 敵対的レビュー（CLI 経由）
1. 変更を加えた状態で "Codexに反論してもらって" と指示
2. 期待結果:
   - 敵対的プロンプトで `codex review --uncommitted "..."` が実行される
   - 設計の妥当性・代替案・障害モードなどの疑義が返される
   - Claude 自身の見解との「合意点」「相違点」が整理される

### テスト 8: 敵対的レビュー（プラグイン経由）
1. "設計の妥当性をCodexに疑ってもらって" と指示
2. 期待結果:
   - `/codex:adversarial-review` コマンドの実行案内または実行
   - 設計疑義・トレードオフ分析が返される

### テスト 9: 構造化出力レビュー
1. "構造化出力でCodexにレビューして" と指示
2. 期待結果:
   - JSON 形式のプロンプトで `codex review` が実行される
   - verdict, findings (severity 別), next_steps が整理されて提示される

### テスト 10: バックグラウンドレビュー（プラグイン）
1. "バックグラウンドでCodexにレビューしてもらって" と指示
2. 期待結果:
   - `/codex:review --background` の実行案内
   - `/codex:status` でジョブ確認方法を案内

### テスト 11: ジョブ管理（プラグイン）
1. `/codex:review --background` 実行後に "/codex:status" を確認
2. 期待結果: 実行中・完了済みジョブ一覧が表示される
