# Test Plan

## トリガーテスト

### Should trigger（発動すべきケース）
- "Codexにも確認して"
- "Codexでチェックして"
- "Codexにセカンドオピニオンをもらって"
- "このコードをCodexにレビューしてもらって"
- "コミット前にCodexに確認してもらって"
- "mainとの差分をCodexに見てもらって"
- "Codexの意見も聞きたい"
- "別のAIにも見てもらって"

### Should NOT trigger（発動すべきでないケース）
- "このコードをレビューして"（Codex 言及なし → code-reviewer スキルが処理）
- "Codex CLIをインストールして"（インストール支援、レビューではない）
- "Codexとは何ですか"（概念説明の質問）
- "コードの問題を直して"（バグ修正依頼）

---

## 機能テスト

### テスト 1: 未コミット変更のレビュー
1. git リポジトリで何か変更を加える
2. "今の変更をCodexにもレビューしてもらって" と指示
3. 期待結果:
   - `codex review --uncommitted "..."` コマンドが実行される
   - Codex の回答が取得される
   - Claude の見解と統合されたフィードバックが返される

### テスト 2: ファイル単体レビュー
1. レビュー対象のファイルを指定
2. "このファイルをCodexにレビューしてもらって" と指示
3. 期待結果:
   - `cat {ファイルパス} | codex exec "..."` コマンドが実行される
   - Codex の回答が取得される
   - Claude の分析と統合されたフィードバックが返される

### テスト 3: ブランチ差分レビュー
1. feature ブランチで作業中の状態
2. "mainとの差分をCodexに確認してもらって" と指示
3. 期待結果:
   - `codex review --base main "..."` コマンドが実行される
   - 差分に対するレビューが返される

### テスト 4: 機密ファイルの確認
1. `.env` や認証情報を含むファイルを指定
2. 期待結果: 外部送信前にユーザーへ確認プロンプトが表示される

### テスト 5: 未ログイン時
1. `codex logout` でログアウトした状態でスキルを実行
2. 期待結果: `codex login` の実行案内が表示される

### テスト 6: git リポジトリ外でのレビュー
1. git リポジトリ外のディレクトリでスキルを実行（`codex review` を使おうとした場合）
2. 期待結果: ファイル単体レビュー（`cat file | codex exec`）への切り替え案内が表示される
