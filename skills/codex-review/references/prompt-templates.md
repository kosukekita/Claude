# Prompt Templates

Codex CLI に渡すプロンプトのテンプレート集。
`codex review` の場合は引数として、`codex exec` の場合はパイプ後の引数として使用する。

## コードレビュー

```
以下のコードをレビューしてください：
- バグや潜在的な問題
- パフォーマンスの改善点
- セキュリティの懸念
- コードの可読性・保守性
日本語で回答してください。
```

**git diff 使用例:**
```bash
codex review --uncommitted "以下のコードをレビューしてください：バグや潜在的な問題、パフォーマンスの改善点、セキュリティの懸念、コードの可読性・保守性。日本語で回答してください。"
```

**ファイル使用例:**
```bash
cat {ファイルパス} | codex exec "以下のコードをレビューしてください：バグや潜在的な問題、パフォーマンスの改善点、セキュリティの懸念、コードの可読性・保守性。日本語で回答してください。"
```

## 論文・学術文章チェック

```
以下の文章を学術的な観点からレビューしてください：
- ロジックの一貫性
- 文法・表現の改善点
- 引用や根拠の妥当性
日本語で回答してください。
```

**使用例:**
```bash
cat {論文ファイルパス} | codex exec "以下の文章を学術的な観点からレビューしてください：ロジックの一貫性、文法・表現の改善点、引用や根拠の妥当性。日本語で回答してください。"
```

## セキュリティ監査

```
以下のコードをセキュリティの観点から監査してください：
- 認証・認可の問題
- インジェクション攻撃（SQLi, XSS等）への脆弱性
- 機密情報の取り扱い
- 依存ライブラリのリスク
日本語で回答してください。
```

## PR前の最終確認（git diff 向け）

```bash
# 未コミット変更の総合レビュー
codex review --uncommitted "このPR前の変更について総合的にレビューしてください。マージしても問題ないか、懸念点があれば日本語で教えてください。"

# ブランチ差分のレビュー
codex review --base main "main ブランチとの差分をレビューしてください。日本語で回答してください。"
```

## 敵対的レビュー（Adversarial Review）

設計判断・トレードオフ・障害モード・暗黙の仮定を疑うレビュー。
`/codex:adversarial-review` コマンドの代替として使用する。

```bash
codex review --uncommitted "以下の変更について敵対的レビューを行ってください：
1. この設計判断は本当に最適か？代替案とそのトレードオフを具体的に示せ
2. この実装が壊れるシナリオは何か？障害モードを列挙せよ
3. 暗黙の前提条件は何か？それが崩れた場合どうなるか
4. スケーラビリティ・保守性・セキュリティで見落としている懸念はないか
5. このアプローチに対する最も強い反論は何か
日本語で回答してください。"
```

**ファイル単体への敵対的レビュー:**
```bash
cat {ファイルパス} | codex exec "以下のコードについて敵対的レビューを行ってください：設計の妥当性を疑い、代替案・障害モード・暗黙の前提・見落とした懸念点を指摘してください。日本語で回答してください。"
```

## 構造化出力レビュー（Structured Output Review）

`/codex:review` の構造化 JSON 出力に相当するフォーマット。
結果を自動パースして severity 別に整理する場合に使用する。

```bash
codex review --uncommitted "以下の変更をレビューし、結果を以下のJSON形式で返してください：
{
  \"verdict\": \"approve | needs-attention\",
  \"summary\": \"総合所見（1-2文）\",
  \"findings\": [
    {
      \"severity\": \"critical | warning | suggestion\",
      \"title\": \"問題の短いタイトル\",
      \"body\": \"詳細説明\",
      \"file\": \"ファイルパス（不明な場合は null）\",
      \"line\": \"行番号または範囲（不明な場合は null）\",
      \"confidence\": \"high | medium | low\",
      \"recommendation\": \"推奨する修正方法\"
    }
  ],
  \"next_steps\": [\"次にすべきアクション1\", \"アクション2\"]
}
JSONのみを返してください。説明文は不要です。"
```
